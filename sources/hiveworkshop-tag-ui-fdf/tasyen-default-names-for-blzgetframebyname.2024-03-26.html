Introduction

This list was generated by reading/interpreting all fdf files from warcraft 3 ptr 1.31 and reading all words in "".
LeaderBoard/Multiboard/TimerDialog do only work when the map created such ones.
LogDialog is only used in SinglePlayer.
Chat Frames are only used in Multiplayer.
Quest Frames are created after the Quest Button was clicked (can be forced with code, first line opens the quest dialog the second closes it)
call BlzFrameClick(BlzGetFrameByName("UpperButtonBarQuestsButton", 0))
call BlzFrameClick(BlzGetFrameByName("QuestAcceptButton", 0))​
If there is no nr next to a name, one uses "0".
For example BlzGetFrameByName("HelpTextArea",0) would return the frame handling the help Text Area.

yes, there is. First one has to either (hide default ui + make the face visible again), or use call BlzEnableUIAutoPosition(false)
JASS: 
call BlzFrameClearAllPoints(BlzGetOriginFrame(ORIGIN_FRAME_PORTRAIT,0))
call BlzFrameSetAbsPoint(BlzGetOriginFrame(ORIGIN_FRAME_PORTRAIT,0), FRAMEPOINT_BOTTOMLEFT, 0.4, 0.3)
Unit portrait is a originframe, ORIGIN_FRAME_PORTRAIT.

One can reach the ChargeBox using the child-Frame api (Warcraft 3 V1.32.6+), but there is no access to the String-Frame displaying the number.
After the command button shows the box the frame is created and than can be reached with.
Lua: 
BlzFrameGetChild(BlzGetOriginFrame(ORIGIN_FRAME_COMMAND_BUTTON, 0), 2)

It is the same for ORIGIN_FRAME_ITEM_BUTTON.

BlzFrameGetChild crashs the game when you try to get an index not set. (out of bounce exception)
First test if BlzFrameGetChildrenCount is bigger than the wanted number.
